# Agent Rule: Release Build & Upload Flow

本规则用于约束代理在发布版本时的最小执行流程，确保 Releases 页面可见且内容完整。

## 固定流程

1. 本地构建  
   - 在仓库本地完成目标平台构建。  
   - 构建产物需包含用户要求的平台与文件命名。

2. 编写发布说明  
   - 根据用户提示与本次提交内容撰写发布说明。  
   - 发布说明必须中英双语、简洁明了。

3. 发布到 Releases  
   - 将 tag 与目标提交对齐后，发布到 `/wild-pig-gun/releases`。  
   - 上传对应平台构建产物，并校验资产列表完整。

## 执行要求

- 在创建或更新 release 前，先校验 tag 是否已存在与其指向提交。  
- 若同名 tag 指向错误提交，需先与用户确认后替换。  
- 发布后必须复核 release 列表可见性、tag 指向和资产数量。
